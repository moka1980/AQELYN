# UX Requirements Matrix

| ID | Requirement | Verification |
|---|---|---|
| UX-001 | Every finding shall have a non-technical summary. | UI Review / Unit Test |
| UX-002 | Every finding shall have an expert-detail expansion. | UI Review / Integration Test |
| UX-003 | Every recommendation shall identify expected effort. | Requirements Test |
| UX-004 | Every recommendation shall identify expected security benefit. | Requirements Test |
| UX-005 | Automated remediation shall require consent unless pre-approved by policy. | Security Test |
| UX-006 | Every finding shall link to evidence. | Traceability Test |
| UX-007 | Explanations shall avoid unnecessary fear-based language. | Content Review |
| UX-008 | Platform shall support home, SMB, enterprise, and expert communication modes. | System Test |
| UX-009 | Users shall understand what data AQELYN used. | UI Review |
| UX-010 | Users shall know what to do next. | UX Acceptance Test |
