AQELYN Query & Discovery Language (SQDL)
Status: Core Architecture
Priority: Critical
Vision
AQELYN should have its own security language.
Not SQL.
Not Elasticsearch syntax.
Not SIEM query languages.
A language designed specifically for cybersecurity.
Users should be able to ask questions naturally.
Examples:
Show Finance laptops
or
Find websites
or
Show all mobile devices
The platform translates this into object queries.
Why SQDL?
Current security platforms require users to learn:
•  SQL
•  KQL
•  SPL
•  Lucene
•  Elasticsearch
•  Vendor-specific languages
Instead, AQELYN provides one consistent language across every module.
Universal Query Model
Everything is queried through objects.

PS-ADR-0030
Search-First Platform
Status: Accepted
Decision
Project  AQELYN  shall  adopt  a  search-first  navigation  model  where  users  can  locate
objects, missions, evidence, policies, and reports through a unified discovery interface.
Rationale
Large  cybersecurity  platforms  become  difficult  to  navigate  as  they  grow.  A  search-first
model improves usability, reduces training requirements, and supports both novice and
expert users.

Examples:

•  Bug Bounty Workspace (authorized targets only).
•  Endpoint Workspace.

•  Cloud Workspace.
•  Compliance Workspace.
•  Home Workspace.
Each workspace combines:
•  Search.
•  Dashboards.
•  Objects.
•  Missions.
•  Queries.
•  Evidence.
•  Reports.
This means every role sees exactly what they need without being overwhelmed by the

I  want  to  formally  add  another  design  rule  that  reflects  everything  you've  emphasized
since the first day:
The platform must scale in complexity without scaling in user effort.
That means:
•  A home user protecting one laptop should need only a few clicks to understand their
security status.

thousands  of  assets  using  the  same  consistent  concepts:  Objects,  Missions,
Evidence, Knowledge, Trust, and Workspaces.

Looking Ahead
The next foundational volume will define the AQELYN AI Orchestration Framework.
Unlike  many  products  that  simply  "add  AI,"  we  will  design  a  framework  where  AI  is
modular,  explainable,  replaceable,  privacy-aware,  and  always  constrained  by  the
evidence model and human oversight. That will allow organizations to use local models,
cloud models, or future AI technologies without redesigning the rest of AQELYN.
