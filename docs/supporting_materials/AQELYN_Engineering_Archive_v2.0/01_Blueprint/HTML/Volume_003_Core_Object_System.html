Chapter 1
Why an Object System?
Traditional security platforms are data-centric.
AQELYN shall be object-centric.
Every entity within the platform shall exist as a first-class object with a unique identity,
lifecycle, relationships, history and evidence.
Objects are the foundation of the platform.
Everything else builds upon them.

Chapter 13
Object Permissions
Access to objects is controlled through Role-Based Access Control (RBAC).
Permissions are assigned to actions (for example: view, modify, approve, delete) rather
than  directly  to  users.  This  makes  the  authorization  model  more  flexible  and  easier  to
audit.

New Architectural Decision
PS-ADR-0001
Title
Object-First Architecture
Status
Accepted
Decision
AQELYN shall use an Object-First Architecture in which every meaningful entity is
represented as a persistent object with identity, relationships, history and evidence.
Rationale
This provides a consistent model across all modules, simplifies integration, and enables
advanced capabilities such as Digital Twin, Knowledge Graph, AI-assisted reasoning and
comprehensive auditing without creating different data models for each subsystem.
One important adjustment before we continue
I would like to make one architectural refinement.
Earlier  we  discussed  "not  using  a  database."  After  thinking  through  the  engineering
implications, I recommend we rephrase that idea.
A reliable platform will still need persistent storage. Instead of saying "no database,"
I suggest we define:
•  The Object Engine is the primary abstraction.
•  The persistence layer is an implementation detail.
That means every module works with objects and never depends directly on a specific
storage  technology.  Whether  we  later  choose  PostgreSQL,  another  database,  or  a
different  persistence  mechanism  is  hidden  behind  the  Object  Engine.  This  keeps  the
architecture clean, portable and realistic while preserving the vision that developers think
in terms of objects—not database tables. I believe this is the strongest long-term design
because it is both technically sound and fully achievable.
